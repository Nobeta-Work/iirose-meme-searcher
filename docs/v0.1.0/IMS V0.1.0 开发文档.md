# IMS V0.1.0 开发文档

## 1. 文档信息
- 项目名称：iirose-meme-searcher（IMS）
- 文档版本：v0.1.0
- 文档日期：2026-03-21
- 关联需求文档：`IMS v0.1.0 需求文档.md`
- 当前状态：开发准备完成（可进入实现）

## 2. 开发目标
- 在 iirose 网页端完成一个可通过远程 JS 导入的前端插件。
- 用户在输入框输入前缀后，能够触发图片搜索。
- 搜索结果以“输入框锚定的非模态悬浮候选条”形式展示。
- 用户点击候选图片后，将裸图片 URL 发送到公屏。
- 提供轻量配置面板，至少支持前缀关键词、搜索接口地址、候选上限配置。
- 提供检索前缀词配置，允许多个前缀词与用户关键词组合查询。
- 保证 v0.1.0 具备基本可用性、稳定性与可回归验证能力。

## 3. 开发范围映射
- 本文档仅覆盖 v0.1.0 开发与测试，不展开后续版本能力设计。
- 默认搜索接口：由安装器注入或用户在配置面板填写。
- 默认候选数量：8。
- 候选数量配置范围：1-20。
- 检索前缀词：支持多值配置。
- 兼容端：iirose 网页端。
- 发送范围：仅公屏。
- 导入方式：远程 HTTPS 单文件 JS，通过 JS 终端安装器片段导入。

### 3.1 开发前冻结结论
- 前缀输入同时支持 `/m关键词` 与 `/m 关键词`。
- 搜索接口地址由配置面板直接维护。
- 检索前缀词在配置中以多值形式维护，搜索时与用户关键词按空格拼接。
- 候选条为输入框锚定的非模态悬浮候选条，不采用独立弹窗方案。
- 候选条默认展示 8 个结果，配置范围为 1-20。
- 候选条关闭或重置条件固定为：前缀被删除、房间切换、发送成功。
- 输入框失焦不是 v0.1.0 的强制关闭条件。
- 发送成功后清空输入框。
- 远程导入形态固定为单文件 HTTPS JS + 一行式安装器片段。

## 4. 开发原则
- 先可观测，再开发功能。任何核心路径开发前，先确认可监听输入、可定位输入框、可定位主页面上下文。
- 先闭环，再增强。优先保证“输入前缀 -> 搜索 -> 展示 -> 点击发送 -> 反馈”的最小闭环。
- 先收敛接口，再写 UI。先定义运行时模块边界与数据模型，再实现候选条与配置面板。
- 先做好去重与销毁，再谈复用。远程脚本必须具备重复导入保护、页面刷新后可重新初始化。
- 每个阶段结束前都做阶段性回归，不把问题堆到最后一轮。

### 5.1 安装器模板占位

```js
(() => {
  const SRC = 'https://<release-domain>/ims.js'
  const frame = document.getElementById('mainFrame')
  const hostWin = frame?.contentWindow || window
  const hostDoc = hostWin.document

  if (hostWin.__IMS_V010_LOADING__ || hostWin.__IMS_V010__) return
  hostWin.__IMS_V010_LOADING__ = true

  const old = hostDoc.querySelector('script[data-ims-version="v0.1.0"]')
  if (old) old.remove()

  const script = hostDoc.createElement('script')
  script.src = `${SRC}?t=${Date.now()}`
  script.dataset.imsVersion = 'v0.1.0'
  script.onload = () => { hostWin.__IMS_V010_LOADING__ = false }
  script.onerror = () => { hostWin.__IMS_V010_LOADING__ = false }
  hostDoc.head.appendChild(script)
})()
```

- 以上内容为开发占位模板，不代表最终发布域名。
- 最终实现可调整细节，但必须保持“一行式安装器片段 + 单文件 HTTPS JS”的导入机制不变。

## 6. 建议模块划分
- `bootstrap`：启动、重复导入保护、主页面上下文定位、生命周期管理。
- `runtime`：输入框定位、输入监听、房间切换感知、页面状态同步。
- `search`：搜索接口请求、结果解析、结果归一化、异常处理。
- `query-builder`：检索前缀词与用户关键词拼接、去重、标准化。
- `ui-candidates`：候选条渲染、更新、清空、横向滚动、点击行为。
- `ui-settings`：轻量配置面板、配置项读写、配置项校验。
- `send`：消息格式组装、公屏发送、发送状态反馈。
- `storage`：本地持久化、默认值回填、配置版本兼容。
- `diagnostics`：开发日志、调试开关、关键状态输出。

## 7. 分阶段开发计划

### 阶段 0：基线摸底与运行时探针
- 目标：确认 iirose 网页端主页面结构、输入框定位方式、消息发送入口、JS 终端导入上下文。
- 核心任务：
  - 确认顶层页面与 `mainFrame` 的关系。
  - 确认脚本最终应运行在顶层还是主界面 frame 内。
  - 确认聊天输入框、发送相关 DOM、房间切换相关标志的最小定位方案。
  - 确认是否能稳定监听输入内容变化。
  - 确认开发期日志输出方式。
- 产物：
  - 页面结构探针记录。
  - 关键 DOM 选择器候选列表。
  - 最小运行时验证脚本。
- 阶段完成标准：
  - 可以稳定在目标上下文执行自定义 JS。
  - 可以稳定获取输入框引用或可替代监听点。
  - 可以确认候选条的锚定容器与注入位置。

### 阶段 1：远程导入与生命周期框架
- 目标：建立远程脚本加载、初始化、去重、销毁的基础能力。
- 核心任务：
  - 实现远程单文件 JS 入口。
  - 实现全局实例锁或版本标记，防止重复初始化。
  - 实现安装器片段，自动定位当前有效上下文并注入脚本。
  - 实现插件初始化流程。
  - 实现基础日志开关。
- 产物：
  - 远程脚本入口。
  - 安装器导入语句草案。
  - 生命周期图与初始化时序说明。
- 阶段完成标准：
  - 同一页面重复执行导入语句不会创建多个实例。
  - 刷新页面后重新导入可以正常工作。
  - 初始化失败时有明确日志输出。

### 阶段 2：输入监听与触发识别
- 目标：完成前缀识别、关键词提取、状态切换。
- 核心任务：
  - 监听输入框内容变化。
  - 识别 `/m` 前缀。
  - 同时兼容 `/m关键词` 与 `/m 关键词` 的解析策略。
  - 解析当前配置中的检索前缀词列表。
  - 将输入状态抽象为：未触发、已触发空关键词、已触发有关键词。
  - 明确候选条显示与隐藏条件。
- 产物：
  - 触发状态机。
  - 关键词解析函数。
  - 输入监听日志样例。
- 阶段完成标准：
  - 输入前缀后能稳定进入搜索态。
  - 删除前缀后能稳定退出搜索态。
  - 房间切换后能正确重置状态。
  - 同时支持 `/m关键词` 与 `/m 关键词` 两种输入形式。

### 阶段 3：搜索服务与结果归一化
- 目标：建立搜索接口调用链路与结果归一化模型。
- 核心任务：
  - 确定搜索接口请求协议。
  - 先将检索前缀词与用户关键词组合为最终查询词。
  - 实现请求发起、超时处理、失败重试入口。
  - 从搜索结果中抽取最小字段：`id/name/keywords/url/enabled`。
  - 对结果做基础去重、URL 有效性校验、候选上限截断。
  - 定义加载中、空结果、失败三类状态。
- 产物：
  - 搜索服务模块。
  - 归一化结果结构定义。
  - 失败与降级策略说明。
- 阶段完成标准：
  - 输入关键词后可返回归一化结果。
  - 请求失败时可进入失败态。
  - 空关键词时不发请求或按设计直接进入空态。

### 阶段 4：候选条 UI
- 目标：完成输入框锚定的候选条 UI。
- 核心任务：
  - 在输入框附近注入候选条容器。
  - 保证候选条通常显示在输入框上方。
  - 支持横向滚动。
  - 支持缩略图与名称展示。
  - 实现加载中、空态、失败态、正常态四种渲染。
  - 确保候选条不抢输入焦点。
- 产物：
  - 候选条组件。
  - 样式注入方案。
  - 候选条状态切换图。
- 阶段完成标准：
  - 候选条可随输入更新内容。
  - 候选条不会阻断用户继续输入。
  - 常见窗口尺寸下可见、可点击、可横向滑动。

### 阶段 5：点击发送闭环
- 目标：打通候选点击到公屏发送的完整闭环。
- 核心任务：
  - 将点击候选映射为裸图片 URL 发送。
  - 对接网页端消息发送入口。
  - 增加发送中防重复点击保护。
  - 发送成功后清空输入框。
  - 展示成功或失败反馈。
- 产物：
  - 发送模块。
  - 成功/失败反馈组件或提示机制。
  - 重复点击保护逻辑。
- 阶段完成标准：
  - 点击候选可向公屏发送裸图片 URL。
  - 成功后输入框清空。
  - 连续点击不会造成明显重复发送。

### 阶段 6：轻量配置面板与本地持久化
- 目标：提供普通用户可用的轻量配置入口。
- 核心任务：
  - 提供配置入口 UI。
  - 支持修改前缀关键词。
  - 支持搜索接口地址配置。
  - 支持维护多个检索前缀词。
  - 支持候选上限配置，范围为 1-20。
  - 明确搜索接口地址输入、校验与持久化策略。
  - 实现默认值回填、非法值校验、本地持久化。
  - 配置修改后即时生效。
- 产物：
  - 配置面板。
  - 配置 Schema。
  - 本地存储读写适配层。
- 阶段完成标准：
  - 默认值正确。
  - 刷新页面后配置仍可恢复。

### 阶段 7：稳定性增强与边界处理

## 9. 开发顺序建议
1. 先做阶段 0 与阶段 1，确认导入、上下文、去重机制。
2. 再做阶段 2，保证输入识别稳定。
3. 随后完成阶段 3 与阶段 4，先把结果正确展示出来。
4. 再做阶段 5，打通点击发送闭环。
5. 然后做阶段 6，补齐配置面板与持久化。
6. 最后完成阶段 7 与阶段 8，集中处理稳定性与回归。

## 11. 回归测试计划

### 11.1 冒烟回归
- 能通过导入语句加载插件。
- 重复执行导入语句不会产生双实例。
- 刷新页面后重新导入可用。
- 输入 `/m` 能进入搜索态。
- 点击候选可发送公屏消息。

### 11.2 功能回归
- 输入 `/m猫猫` 能触发搜索。
- 输入 `/m 猫猫` 能触发搜索。
- 空关键词时展示空态。
- 有关键词时展示候选条。
- 候选条默认显示 8 个结果。
- 候选上限改为 1、8、20 时均能生效。
- 配置面板可修改前缀。
- 配置面板可修改搜索接口地址。
- 配置面板可修改多个检索前缀词。
- 配置面板可修改候选上限。
- 搜索接口地址保存后可生效。
- 检索前缀词配置后，查询词拼接结果正确。
- 配置刷新后仍保留。
- 发送成功后输入框被清空。
- 发送失败时有反馈。

### 11.3 UI 回归
- 候选条显示在输入框附近，通常位于输入框上方。
- 候选条不抢输入焦点。
- 候选条支持横向滑动。
- 候选图、名称显示完整度可接受。
- 加载态、空态、失败态可区分。
- 配置面板与候选条不会混用区域。
- 常见窗口尺寸下候选条不遮挡核心发送控件。

### 11.4 兼容与状态回归
- 页面首次加载后导入正常。
- 切换房间后候选条状态重置正常。
- 删除前缀后候选条隐藏正常。
- 发送成功后候选条关闭或重置正常。
- 输入内容快速变化时候选条不会明显错乱。
- 重复快速点击候选不会造成明显重复发送。
- 页面局部刷新或输入框重新挂载后可恢复监听。

### 11.5 网络与异常回归
- 搜索接口请求超时时进入失败态。
- 搜索接口返回空结果时进入空态。
- 返回结果中包含无效 URL 时被过滤或降级处理。
- 搜索接口访问失败时仍能给出可感知反馈。
- 异常日志不会导致页面报错中断主流程。

### 11.6 导入与发布回归
- 安装器片段可在目标上下文注入远程脚本。
- 远程脚本为 HTTPS。
- 响应内容类型可执行。
- 页面刷新后脚本视为卸载，再次导入正常。
- 远程脚本版本标记或实例标记生效。
- `mainFrame` 存在与不存在两种情况下，安装器都能给出可预期结果。

## 14. 缺陷分级建议
- P0：无法导入、无法初始化、无法发送、页面主功能被破坏。
- P1：无法搜索、候选条无法展示、配置失效、明显重复发送。
- P2：布局异常、反馈不清晰、个别状态不一致。
- P3：文案、样式、低频边角问题。

## 15. 发布前检查清单
- 需求文档与开发文档一致。
- 安装器导入语句已定稿。
- 远程 JS 地址可访问。
- 搜索接口地址已定稿并可用。
- 候选条与配置面板都可用。
- 默认配置正确。
- 候选数量配置范围正确。
- 发送行为符合“裸 URL + 公屏”约束。
- 回归测试完成并记录。
- 已知风险已记录。

## 16. 已知风险
- Bing 搜索结构可能变化。
- iirose 网页端 DOM 结构可能变化。
- 远程脚本托管域名可用性可能波动。
- 非官方前端注入方式可能受页面更新影响。

## 17. 结论
- v0.1.0 开发应按“运行时探针 -> 导入框架 -> 触发识别 -> 搜索服务 -> 候选条 -> 点击发送 -> 配置面板 -> 稳定性与回归”的顺序推进。
- 回归测试不是收尾动作，而是每阶段出场条件的一部分。
- 若阶段 0 或阶段 1 发现网页端关键注入点不可稳定获取，应立即暂停功能开发，先回到可行性验证。
